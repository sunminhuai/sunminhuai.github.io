基于Hexo+coding建立的个人博客

一些废话

原博客sunminhuai.xyz由于域名到期、腾讯cloud studio升级、操作失误等因素导致无法访问+博文丢失，还好保存了一份数据库文件以便日后逐步恢复旧博文

博客建站技术

博客建站技术非常多，本文仅将个人已知的方法归为博客平台、动态网页、静态网页3类。

博客平台

  • CSDN，简书，wordpress等传统博客平台，大部分平台免费且有广告但访问量大适合推广，少部分平台收费但访问量很少
  • 知乎、语雀、gitbook、baklib等新兴内容平台，免费且访问量不错

需要服务器的动态网页

  • 最常见的就是一般的建站技术，即主机+建站程序，经过本人实践证实想要好的建站体验还是要花钱，也有少部分低价甚至免费主机但体验不好，免费的要么不稳定要么需要不断做推广续期，而国内服务器的备案还超级麻烦，建站程序有Wordpress，Typecho等

    收集到的一些免费主机

    • AWS和谷歌云的免费使用一年服务，但需要支持VISA的信用卡，折腾起来挺麻烦的（AWS中国区不支持个人申请貌似需要企业执照，淘宝mastercard卡是1块）
    • 三丰云，免费虚拟主机和云服务器
    • 阿贝云，免费虚拟主机和云服务器
    • 萝莉云，很便宜也有免费的
    • 主机屋，免费空间
    • 主机蛋，免费空间
    • 综合免费信息，免费空间网
    • 慧选，免费主机
    • 硅云，1元主机
    • 热铁盒
    • 沐云
  • 另一种方法就是利用cloud studio的动态部署功能可以实现免费建站，旧站点sunminhuai.xyz就是使用该方法建立的wordpress站点，但站点会偶尔访问不了或者访问很慢，而目前cloud studio被腾讯升级后貌似已经没有动态部署功能了

不需要服务器的静态网页

  • 主要就是框架+托管的方式，框架有Hexo、Jekyll、Vuepress等等，托管平台有github、coding、码云（免费版不能自动部署，pages pro收费）等，该方法需要自己搭建环境，较为繁琐，但可拓展性很高，部分涉及写作的命令、代码已经可以使用插件代替，如hexoeditor、hexoblogclient等，尤其是hexo的插件、主题市场非常丰富，需要吐槽的是github的国内下载速度超慢
  • 作为该方法的优化，出现了客户端+框架+托管的方式，如Gridea+coding的方式，该方法非常简单快速，但扩展性不足，需要吐槽的是其自带的编辑器bug不少；还有就是本博客使用的hexoblogclient+hexo+coding的方式，可以实现一键部署、博文本地管理等功能

搭建博客

需求：尽可能免费或者低价，稳定简便，本地存储，书写优雅
方案：采用hexoblogclient编写并本地存储+一键部署+Coding托管的方式
安装步骤：

  • 安装git，安装nodejs
  • git bash here
npm config set registry=http://registry.npm.taobao.org/
#根据个人需要设置淘宝镜像加速下载，本人更换后下载并没有变快
npm config set registry https://registry.npmjs.org
#恢复原镜像
npm install -g hexo-cli
npm init
#安装hexo
hexo g
hexo s
#产生静态文件并运行服务器，打开http://localhost:4000能看到博客就说明安装成功
git clone https://github.com/blinkfox/hexo-theme-matery.git
#下载hexo-theme-matery主题
  • 安装hexo-theme-matery主题：在_config.yml中更改theme:landscape为theme:hexo-theme-matery
  • 在coding免费团队版中建立项目，初始化，开启page服务，具体参考这里
  • 在_config.yml中找到deploy，更改配置
deploy:
  type: git # 必须填git
  repo: # coding仓库的http地址
  # repo示例 https://e.coding.net/用户名/项目名/仓库名.git
  branch: master # 分支
  message: blog update # 每次更新时提交的备注
  • 测试部署博客
npm install --save hexo-deployer-git
hexo clean
hexo g -d 
#若coding仓库中代码更新并可以正常访问page，则部署成功，输入d后会要求填用户名和密码，需要注意coding团队版是邮箱和密码登录而非用户名
  • 在_config.yml中更改博客设置，更多设置参考官方文档
# Site
title: Hexo         # 此处改为你站点的标题
subtitle:           # 此处改为你站点的副标题
description:        # 此处改为你站点的说明
keywords:           # 此处改为你站点的关键字
author: John Doe    # 此处改为你的名字
language:           # 此处改为 zh-CN
timezone:           # 此处改为 Asia/Shanghai
  • 主题安装后需要在\hexo\source\下分别建立\about、\categories、\tags等文件夹，并分别在里面建立\index.md文件
---
title: "about"
type: about
layout: "about"
---
# 其它分别以categories、tags等替换about，若还有其他页面找不到就按相同方法建立index即可
  • 在主题的_config.yml中更改主题设置，具体参考主题文档
    • 更改图片替换文件夹文件即可
    • 更改图标在E:\博客\hexo\themes\hexo-theme-matery\source\libs\awesome\css\all.css和图标网站中查询
    • 评论系统：使用valine，不需要登陆，主题内置，配置简单
    • 更改梦想栏为一言，参考博文，保留引用来源参考这里
<script src="https://cdn.jsdelivr.net/npm/bluebird@3/js/browser/bluebird.min.js"></script>

  • 添加关于小站：思路就是通过ejs模板建立新页面；先在主题配置文件中的about下添加二级菜单关于小站，指向/about，图标用fab fa-internet-explorer；然后打开theme/matery/layout/about.ejs文件，大约在13行有一个<div class="card">标签，在之前新增如下代码，之后在/source/about/index.md下面写上内容即可
<div class="card">
    <div class="card-content">
        <div class="card-content article-card-content">
            <div class="title center-align" data-aos="zoom-in-up">
                <i class="fab fa-internet-explorer"></i>&nbsp;&nbsp;<%- __('关于小站') %>
            </div>
                <div id="articleContent" data-aos="fade-up">
                    <%- page.content %>
                </div>
        </div>
    </div>
</div>
  • 添加个人简历：先在主题配置文件中的about下添加二级菜单个人简历，指向/resume，图标用fas fa-portrait以修改工具栏图标，并新建/hexo/source/resume/index.md，foront-matter全部改为resume；然后新建theme/matery/layout/resume.ejs文件，新增以上同样的代码并改为个人简历，并更改对应的图标名以修改网页内图标，之后在/source/resume/index.md下面写上内容即可
  • 更改友链为常用工具：跟以上类似，在主题配置文件中将工具栏图标改为fas fa-toolbox，在\hexo\themes\hexo-theme-matery\layout\friends.ejs中修改网页图标，在\hexo\source\_data\friends.json中修改需要的工具链接
  • 添加html自定义页面
    首先在站点目录下的source文件夹下新建aboutme文件，文件名可自定义，然后编写一个index.html放入aboutme文件夹下，然后在主题配置文件下的导航配置信息添加下面的配置：
About:
    url: /
    icon: fas fa-address-card
    children:
      - name: 关于我
        url: /about
        icon: fas fa-user-circle
      - name: Another    #这是新添加的，在原有配置基础上添加
        url: /aboutme
        icon: fa fa-user-secret

然后在站点配置文件下，找到skip_render，在后面添加属性，如下：

skip_render: aboutme/**  # 其意思为在对文件进行渲染时跳过aboutme文件下的所有文件
  • 一键部署：测试了hexoeditor(bug比较多)，hexoclient，hexoblogclent（基于hexoclient二次开发，界面更友好）三款插件，最终选择hexoblogclient
