图形界面交互式机器学习：DIGITS搭建

废话

因为好想用DIGITS的图形化操作界面来搞机器学习，不想写代码啊，所以决定折腾一个；没想到这一折腾就好几天啊，不断的出问题，不断的解决，折腾成功后还喜得流感；刚开始想用官方教程的Linux来搞，就想到了win10的Ubuntu子系统（只有一个1050神州本子），不用虚拟机或者双系统，确实很好用，配合xming就可以实现图形化桌面（DISPLAY=:1 xfce4 -session），然而进一步安装发现折腾显卡驱动就很麻烦，而且各种基础操作也不熟悉，装个输入法都麻烦，最大的问题还是装好后与windows的文件交互，所以果断转向windows的DIGITS，没想到又进入了更多的坑。。。

windows版的DIGITS只支持caffe（Linux上还支持tensorflow等），而且caffe主要有三个版本（branch）：BVLC（伯克利），微软和intel，而只有BVLC版能很好的支持NVIDIA的CUDA（即用GPU），因此想用GPU的只能装这个了（虽然查了下我的这颗本子1050不受官方CUDA支持。。。但看到论坛上有人说不支持的显卡也成功过也就不管了）；但我还是太年轻啊，没想到caffe的安装这么复杂，各种编译错误搞的头都大了。。。不过还好，最后还是成功了，玩溜了以后再搞搞AutoKeras的自动学习（比起收费的谷歌AutoML，爽）

PS：图形化学习方面，还有中科院开源了一个EasyML，也是在浏览器中运行，但看了下界面好丑（貌似是基于java的），而且网上报告的问题也挺多的；还有一些软件提供workflow的操作方式，如KNIME、Rapidminner、SPSSModeler等，但貌似自定义程度低，算法丰富度等方面也不足；自动化学习方面，百度的AutoDL看了下貌似只是接口调用，貌似自定义程度很低；还有第四范式、oneclick、国外的几家在做，但要么要收费要么还没正式开放使用，有名气的开源项目就有AutoKeras和H2O，而H2O官网又显示收费，没太搞懂，以后有时间再研究

搭建过程

硬件环境为神船笔记本，win10 64位，i5 7400 , GTX1050

主要参考资料：
视频教程（需翻墙）：https://www.youtube.com/watch?v=SeCE757egcE&feature=youtu.be
DIGITS官方教程：https://github.com/NVIDIA/DIGITS/blob/master/docs/BuildDigitsWindows.md
caffe官方教程：https://github.com/bvlc/caffe/tree/windows#remark
日本博客（需翻墙，重点推荐）：https://pid123.blogspot.com/2018/06/nvidia-digits6-windows.html
国内博客：http://www.cnblogs.com/qwangxiao/p/9185040.html
搭建思路
Windows下搭建DIGITS最主要的就是搭建caffe，而caffe可以自己编译也可以直接下载编译好的二进制文件（日本大神博客）；然而坑爹的是caffe的官方下载链接尽然打不开？WTF；还有坑爹的是caffe的安装是conda命令，而digits的安装是pip命令，很容易一不小心就装错环境了

环境配置（必选，无为自主编译时必选）
Visual Studio 2015：使用免费社区版和专业版均可；不要看官网说ninja好用就装VS Code去搞ninjia，而且ninja的安装总是有问题，反正我是没搞好；不要装最新版，官方代码里只支持了2015，当然你会自己改也可以；需要注意的是安装时候应该勾选C++，之后需要打开一个sln文件（CUDA安装的文件里有sample）来测试一下，还有激活码一枚：HMGNV-WCYXV-X7G9W-YCX63-B98R2

*Python 2.7：推荐用Anaconda的发行版，可以装最新的anaconda（python3.7），安装后创建一个2.7的环境并激活（命令：conda activate 环境名 ）再在里面安装；不推荐用python2.7的anaconda，因为2.7到2020年就结束支持了，现在已经19年了。。。

*CUDA8和对应的cuDNN：对于菜鸟来说还是老老实实用CUDA8吧，看大神们用CUDA10折腾成功了，而我折腾失败了。。。

CMake：官网下载安装就行

DirectXSDK：选择安装；官方并没提到要安装，但实际测试CUDA的SAMPLES时候会编译通不过，不过我一看不通过，就不再管它了（https://blog.csdn.net/qq_31610789/article/details/80806815）

其它请查看参考资料

思路1：编译caffe
不得不再次吐槽下caffe页面上Prebuilt binaries的下载连接竟然挂了。。。

配置build_win.cmd：可以参考YouTube的视频，改几个参数就行

编译错误
错误1：官方提供的安装命令会报错:
conda install –yes cmake ninja numpy scipy protobuf==3.1.0 six scikit-image pyyaml pydotplus graphviz
改为
conda install –yes cmake ninja numpy scipy six scikit-image pyyaml pydotplus graphviz
conda install protobuf
安装两次
错误2
CMake Error atCMakeLists.txt:143 (add_dependencies):
The dependency target “pycaffe” oftarget “pytest” does not exist.

ImportError: Nomodule named ‘numpy’
– Could NOT findNumPy (missing: NUMPY_INCLUDE_DIR NUMPY_VERSION) (Required is at least version”1.7.1”)
解决：两个错误都是由于numpy没有正确安装导致的（如应该安装在anaconda的环境而非系统默认python环境），重新安装numpy

错误3
C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V140\Microsoft.CppCommon.targets(171,5): error MSB6006: “cmd.exe” 已退出，代码为 1。
解决：是CUDA版本不兼容导致的，VS2015貌似只能用CUDA8

错误4
找不到C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V120\BuildCustomizations\CUDA 8.0.props
编译出错的问题
解决：测试CUDA自带的sample时候发生的，也可以不用解决；原因是Visual Studio 2015的安装路径中缺少CUDA 8.0.props，将 C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v8.0\extras\visual_studio_integration\MSBuildExtensions
拷贝到 C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V120\BuildCustomizations即可

编译后测试
编译成功后exe的位置在C:\Users\自己的文件夹\caffe\build\tools\Release

打开命令行进行测试

conda activate py27
import caffe
quit()
cd C:\Users\你的文件夹\caffe\build\tools\Release
caffe -help
如果没有报错即说明caffe安装成功

思路2：直接用编译好的二进制文件（推荐）
前面已经说过caffe官方提供的链接已经挂了，从MD文件中提取下载链接用下载器也下不了，这就是传说中的死链？然而，说起死链就不得不提迅雷了。果然，冲个迅雷会员，死链下载也是分分钟的事。以下是py27 GPU版的二进制文件：

https://pan.baidu.com/s/1MGye2Sa6Ow58dQrButWVhw
提取码：mdpl

使用这种思路只要按照那个日本大神的博客进行安装即可，需要注意的是所有操作都在powershell中进行

关于启动：需要在克隆了digits的根目录下开启CMD，激活2.7的环境，然后再 python -m digits；否则不会启动
